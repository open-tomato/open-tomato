      Creates and starts an Express HTTP service.

      Startup sequence:

      1. Parse and validate config with Zod — throws on invalid input.
      2. Start each dependency in array order. If any start() rejects, the factory aborts: in test mode (NODE_ENV=test) the error is re-thrown; in production process.exit(1) is called.
      3. Apply the standard middleware stack (helmet, cors, body parser, rate-limit, pino-http).
      4. Build the ServiceContext (logger, auth middleware, dep/client accessors).
      5. Mount built-in routes (GET /health).
      6. Apply plugins in array order.
      7. Call config.register(app, ctx) to mount service-specific routes.
      8. Attach the error handler as the last middleware.
      9. Listen on config.port (or an ephemeral port when NODE_ENV=test).
      10. Register SIGTERM/SIGINT shutdown handlers (skipped in test mode).

      Test-mode behaviour: When NODE_ENV=test, the server binds to port 0 (OS-assigned ephemeral port) and process.exit is never called, making the factory safe to use inside unit and integration tests.

        Service configuration. See ServiceConfig for all options.

        • Optionalauth?: { introspectUrl: string }

          Session introspection configuration. When provided, requireAuth and optionalAuth validate sessions against introspectUrl. When omitted, both middleware are no-ops.

        • Optionalbody?: { limit: string }

          Request-body parser configuration. When omitted, defaults to '1mb' limit.

        • Optionalclients?: Dependency[]

          HTTP clients managed alongside regular dependencies. Accessible via ServiceContext.clients.get(client).

          Default

          []
        • Optionalcontrol?: ControlConfig

          Operator control-plane configuration. When enabled, mounts /_control/* routes protected by secret. See ControlConfig for field descriptions.

        • Optionalcors?: { origins: string[] }

          CORS configuration. When provided, the CORS middleware allows the listed origins. When omitted, cross-origin requests are denied (origin: false).

        • Optionaldependencies?: Dependency[]

          Managed dependencies started before register is called and stopped (in reverse order) when stop() is invoked.

          Default

          []
        • Optionalplugins?: ServicePlugin<{ app: Application } & ServiceContext>[]

          Service plugins applied in array order after middleware and built-in routes but before register.

          Each plugin's register method receives { app, ...ctx } — the Express application merged with the full ServiceContext — so plugins can mount routes and access ctx.logger, ctx.deps, etc.

          Default

          []
        • Optionalport?: number

          TCP port the HTTP server listens on.

          Default

          3000
        • OptionalrateLimit?: { max: number; windowMs: number }

          Rate-limiting configuration applied globally via express-rate-limit. When omitted, defaults to 100 requests per 60 s window.

        • register: (app: Application, ctx: ServiceContext) => void | Promise<void>

          Route/middleware registration callback. Receives the Express app and a fully-constructed ServiceContext. May return a Promise — createService will await it.

        • serviceId: string

          Unique identifier for this service — used in logs and health endpoints.

        • Optionalshutdown?: { drainTimeout: number }

          Graceful-shutdown configuration. When omitted, a 10 s drain timeout is used.

      Returns Promise<ServiceHandle>

      A ServiceHandle containing the Express app and a stop() method that gracefully shuts down the server and stops all dependencies in reverse order.

      Example

      const { app, stop } = await createService({
        serviceId: 'my-service',
        register(app, { logger }) {
          app.get('/hello', (_req, res) => res.json({ hello: 'world' }))
        },
      })
