    Function loadConfig

    • loadConfig(options: LoadConfigOptions): Promise<ResolvedConfig>

      Loads, validates, and resolves the full application configuration.

      Steps performed:

      1. Read and schema-validate the core config YAML at options.configPath.
      2. Apply environment variable overrides — first from process.env, then from options.envOverride (which takes final precedence).
      3. Run semantic validation (bot token resolvability, hook contracts).
      4. If options.hatSources is provided and non-empty, load and merge the hat collections left-to-right; the merged result is attached as hatCollection on the returned object.

      Parameters

      • options: LoadConfigOptions

        Configuration loading options.

        Options accepted by the loadConfig public API entry point.

        • configPath: string

          Path to the core config YAML file.

        • OptionalenvOverride?: Record<string, string>

          Optional env-var key/value pairs that take final precedence over both the config file and process.env.

        • OptionalhatSources?: string[]

          Optional list of hat collection sources to load and merge left-to-right. Each entry is either a file path or a builtin:<name> specifier.

      Returns Promise<ResolvedConfig>

      The fully resolved configuration.

      Throws

      When configPath does not exist.

      Throws

      When configPath contains invalid YAML.

      When schema validation fails.
