# UI Skeleton — Phase 3+ Plan

## Context

Phases 1 (particles + 18 atoms) and 2 (15 molecules) are complete. The empty placeholder barrels under [src/organisms/](./src/organisms/), [src/templates/](./src/templates/), and [src/providers/](./src/providers/) need the same six-file treatment that produced the existing layers. This plan covers everything left to bring the skeleton to feature complete, plus the leftover backlog in [NEXT-ITERATIONS.md](./NEXT-ITERATIONS.md).

Tasks are listed as individual `- [ ]` checkboxes in execution order so [ralph.sh](./ralph.sh) can iterate them one at a time. Each task is self-contained given [PROMPT.md](./PROMPT.md) + this file in context. **Do not edit this file once execution begins** — copy it to `PLAN_TRACKER.md` and tick boxes there per the AGENTS.md plan-tracker convention.

## Goals

- Roll out organisms (17), templates (5), and the Direction provider end-to-end.
- Extend cardinal layer-import rules and ESLint guards to the new layers.
- Close the deferred items in [NEXT-ITERATIONS.md](./NEXT-ITERATIONS.md).
- Keep the per-layer authoring shape consistent across phases.

## Non-goals

- `src/pages/` — out of scope this iteration; placeholder barrel stays.
- Publishing `registry.json` externally.
- Workspace integration of sibling packages beyond items #6 / #9.
- Rewriting any Phase 1 / Phase 2 atom or molecule.

- **Canonical molecule:** [src/molecules/Alert/](./src/molecules/Alert/) — six-file layout, slot vocabulary, lookup-table propagation.
- **Canonical organism (this plan):** Accordion (composes Collapsible molecule; discriminated `type: 'single' | 'multiple'`).

### Accessibility verification for portal-based organisms

`bun run test` runs `jest-axe` per component. For portal-based organisms (Dialog, AlertDialog, Drawer, DropdownMenu, Menubar, Sonner, Command, and the Sheet template), the test MUST call `await axe(document.body)` rather than `await axe(container)` — Radix portals into `document.body`, and a container-scoped axe scan misses the portaled content.
