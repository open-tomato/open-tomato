# Progress notes for ui-skeleton iteration 1

General findings for future tasks (broadly applicable; not task-specific):

- Sibling shared packages live under `packages/shared/<name>/`, not `packages/<name>/`. From `packages/ui-skeleton/` the relative path is `../shared/<name>`. Use this when rewriting `workspace:^` → `file:` links.
- Package names of the relevant siblings: `@open-tomato/eslint-config` and `@open-tomato/typescript-config`.
- `@open-tomato/eslint-config` already depends on `@open-tomato/typescript-config` via `file:../typescript-config` (precedent for the file-link approach).
- `@open-tomato/eslint-config` exposes subpath exports `./base`, `./next`, `./react` — choose the right one in `eslint.config.mjs`.
- `@open-tomato/typescript-config` exposes subpath exports `./base`, `./react`, `./next` (note the JSON filename is `nextjs.json` but the export key is `./next`).
- `@open-tomato/eslint-config` already lists `eslint-plugin-storybook` as a dep, so Storybook lint integration does not require adding it to ui-skeleton.
- Root `packages/` directory does not have a top-level `package.json` workspaces entry covering `ui-skeleton` this iteration — keep it standalone and use `file:` links.
- Git repository root is `packages/` (not the monorepo root above it). Paths in `git status`/`git diff` are relative to `packages/` (e.g. `ui-skeleton/...`). `cd` above `packages/` is not a git repo.
- `PLAN_TRACKER.md` is the working checklist that gets edited every task — expect it to show as modified in `git status`; this is bookkeeping, not a dirty tree.
- Bun 1.3+ writes a text-JSON `bun.lock` (not the older binary `bun.lockb`). It belongs at the ui-skeleton package root and should be committed.
- `bun install` may warn `Blocked 1 postinstall. Run \`bun pm untrusted\` for details.` — this is informational, not a failure. Bun blocks postinstalls from untrusted packages by default.
- `lucide-react` is at major version 1.x (1.16.0+ at time of writing), not the older 0.x range. If a task or doc references an older 0.x version range, treat it as outdated and pin to ^1.x.
- `bun add <pkg>` updates `package.json` automatically and reorders keys alphabetically — expect dependency blocks to be sorted after each add.
- Radix UI primitive packages are mostly at major v1.x, but a few are at higher majors (e.g. `@radix-ui/react-label` is v2.x). Don't assume a uniform major across the `@radix-ui/*` namespace when pinning.
- `bun add` of multiple packages in a single command works and is faster than per-package adds; the lockfile is updated once at the end.
- Tailwind v4 ships as `tailwindcss@^4` (resolves to 4.3.x) with companion packages `@tailwindcss/postcss@^4` (PostCSS plugin) and `@tailwindcss/vite@^4` (Vite plugin). The Vite plugin is the preferred integration for Vite-driven builds; the PostCSS plugin covers tooling that runs PostCSS independently (e.g. Storybook in some configs). `postcss` itself is still needed as a dev dep.
